# Stacks

## Overview

A stack is a linear data structure that follows the Last In First Out (LIFO) principle.

## Key Concepts

- **LIFO**: Last element added is the first one removed
- **Push**: Add element to top
- **Pop**: Remove element from top
- **Peek/Top**: View top element without removing
- **isEmpty**: Check if stack is empty

## Operations

| Operation | Time Complexity |
|-----------|----------------|
| Push      | O(1)           |
| Pop       | O(1)           |
| Peek      | O(1)           |
| Search    | O(n)           |

## Implementations

1. **Array-based**: Fixed or dynamic size
2. **Linked List-based**: Dynamic size

## Common Applications

- Expression evaluation (infix, postfix, prefix)
- Backtracking algorithms
- Function call stack
- Undo/Redo operations
- Browser history
- Balanced parentheses checking

## Important Techniques

1. **Monotonic Stack**: Maintain increasing/decreasing order
2. **Two Stacks**: Simulate queue or solve specific problems
3. **Min/Max Stack**: Track minimum/maximum in O(1)

## Common Problem Patterns

- Parentheses matching
- Next greater/smaller element
- Expression evaluation
- Stock span problems
- Histogram problems
